What is Jinja?

Jinja is a powerful templating engine for Python. It allows you to generate dynamic text files, usually HTML, while keeping the logic separated from the layout. Jinja is often used inside web frameworks, but it can also be used completely standalone for any type of templating.

Jinja syntax is expressive and readable, enabling clean page structures without mixing large amounts of Python logic directly into the file.

Loading a Template in Python

Below is the simplest example of loading and rendering a template using only Python and Jinja.

from jinja2 import Environment, FileSystemLoader

# Load templates from the 'templates' directory
env = Environment(loader=FileSystemLoader("templates"))

# Select template file
template = env.get_template("example.html")

# Render output with variables
html_output = template.render(name="Ana", age=24)

# Save result
with open("output.html", "w") as f:
    f.write(html_output)

Passing Variables to the Template

All variables provided in template.render() become accessible inside the Jinja file using double curly braces.

Python

context = {
    "name": "Ana",
    "country": "France",
    "studying": True
}

html = template.render(context)

Jinja

<p>Name: {{ name }}</p>
<p>Country: {{ country }}</p>

{% if studying %}
    <p>She is studying.</p>
{% else %}
    <p>She is taking a break.</p>
{% endif %}

Output

Name: Ana
Country: France
She is studying.

Lists and Dictionaries

You can pass any Python structure to a Jinja template: lists, dictionaries, or even objects. They can be iterated and accessed naturally.

Python

user = {
    "name": "Ana",
    "projects": [
        {"title": "Tutorial Website", "status": "ongoing"},
        {"title": "FIR Filter SSE", "status": "done"},
        {"title": "FPGA Project", "status": "ongoing"}
    ]
}

html = template.render(user=user)

Jinja

<h2>Projects of {{ user.name }}</h2>

{% for p in user.projects %}
    <p><strong>{{ p.title }}</strong> — {{ p.status }}</p>
{% endfor %}

Conditionals and Loops

Jinja supports if, elif, else, and for structures. They enable dynamic rendering based on the data passed from Python.

If / Else

{% if age >= 18 %}
    You are an adult.
{% else %}
    You are not an adult.
{% endif %}

For Loop

{% for fruit in fruits %}
    {{ fruit }}
{% endfor %}

Filters

Filters allow you to transform values in-line using the pipe symbol. Jinja includes a wide range of built-in filters.

Uppercase

{{ name | upper }}

Example

name = "ana"
                output = ANA

Rounding

{{ price | round(2) }}

Example

price = 12.3485 
                output = 12.35

Macros

Macros are reusable template functions. They help avoid repeating HTML structures.

{% macro card(title, text) %}
<div class="card">
    <h3>{{ title }}</h3>
    <p>{{ text }}</p>
</div>
{% endmacro %}

{{ card("Introduction", "Welcome to the Jinja tutorial!") }}
{{ card("Python", "Learn how to connect Python with templates.") }}

Template Inheritance

Template inheritance allows you to define a base structure and extend it from other templates.

Base Template

<html>
<head>
    <title>{% block title %}{% endblock %} – Tutorial</title>
</head>
<body>
    <header><h1>My Tutorial Website</h1></header>
    <main>
        {% block content %}{% endblock %}
    </main>
</body>
</html>

Child Template

{% extends "base.html" %}

{% block title %}Jinja{% endblock %}

{% block content %}
<h2>Learning Jinja</h2>
<p>Examples and essential concepts.</p>
{% endblock %}
