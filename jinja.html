Introduction to Jinja

Jinja is a modern and flexible templating engine widely used in Python frameworks such as Flask. It allows you to embed dynamic content into HTML, making it easy to combine data and interface without mixing everything into a single file.

Its syntax is simple and expressive, making template logic readable while still keeping the structure of your pages clean.

Variables

Inside a Jinja template, you can display values passed from Python simply by placing the variable name between double curly braces:

{{ user }}

Output (if user = "Ana"):

Ana

You can also access dictionary keys, object attributes, and list indices directly:

{{ person.name }}
{{ product["price"] }}
{{ items[0] }}

Control Structures

Jinja supports logical structures such as conditionals and loops. These are enclosed in {% ... %} tags and allow you to create dynamic sections of HTML based on the data you receive.

If / Else

{% if age >= 18 %}
    You may access this website.
{% else %}
    Access denied.
{% endif %}

Output (if age = 20):

You may access this website.

For Loops

Loops make it easy to iterate through lists and display repeated HTML elements.

{% for item in fruits %}
    {{ item }}
{% endfor %}

Output (if fruits = ["apple", "banana", "grape"]):

apple
banana
grape

Filters

Filters allow you to modify values directly in the template. They are written using the pipe (|) symbol and are very useful for formatting text, numbers, or lists.

Uppercase text

{{ name | upper }}

Output (if name = "ana"):

ANA

Rounding numbers

{{ price | round(2) }}

Output (if price = 5.6789):

5.68

Macros

Macros work like small functions inside templates. They are useful for repeating patterns of HTML without duplicating code.

{% macro greet(name) %}
    Hello, {{ name }}!
{% endmacro %}

{{ greet("Ana") }}

Output:

Hello, Ana!

Blocks and Template Inheritance

One of Jinja’s most powerful features is template inheritance. You can create a base layout (for example, a shared header and footer) and extend it in other pages without rewriting the entire structure.

In your child template, you extend a base file like this:

{% extends "base.html" %}

{% block content %}
    
Welcome to the website!

{% endblock %}

This approach keeps large projects organized, avoids duplicated markup, and makes updates much easier.
